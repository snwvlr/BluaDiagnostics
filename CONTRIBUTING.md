## Como a gente trabalhou

- As decisões técnicas (escolha do modelo, persona, riscos clínicos,
  etc.) foram discutidas em grupo antes de virar código.
- Quem mexeu em arquivo de outra área avisou o grupo antes pra evitar
  conflito.
- Antes de subir qualquer coisa, conferimos que **nenhuma API key**
  aparecia no commit (o `.gitignore` cobre `.env`, `*.key`, etc.).
- O notebook foi testado no Google Colab por mais de um integrante pra
  garantir que rodava no ambiente padrão da disciplina.

## Se você é nosso professor avaliando o trabalho

- Documentação completa está no [README.md](README.md).
- A entrega final consolidada está em [entrega.txt](entrega.txt).
- O notebook é executável no Colab (instruções no cabeçalho dele).
- Qualquer dúvida sobre alguma decisão técnica, o README explica os
  porquês (escolha do modelo, persona, mitigação de riscos).
